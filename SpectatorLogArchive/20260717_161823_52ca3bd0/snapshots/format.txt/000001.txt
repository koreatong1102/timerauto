SpectatorLog Format Reference
==============================

Folder layout
  match/   - round state and per-event data
  blue/    - blue player data
  red/     - red player data

--- match/ ---

round_time.txt
  Single value: elapsed seconds in the current round (F2).

round_state.txt
  Single value: current MatchState name. (Invalid, MatchIntro, RoundIntro, RoundBreak, RoundFight, RoundKnockdown, RoundKnockout, RoundFoul, RoundDisqualified, Results, End, Cancel)

round_number.txt
  Single value: 1-based round index.

round_total.txt
  Single value: total number of rounds in the match.

damage_events.txt  (tab-separated, one row per hit)
  round_time  final_damage  counter_mult  corner  hand  screen_x  screen_y  world_x  world_y  world_z  punch_type  damage_type  weak_point
  round_time  - elapsed seconds at the moment of impact (F2)
  final_damage- damage after all multipliers (F2)
  counter_mult- counter-punch damage multiplier applied to this hit (F2); 1.00 means no counter bonus
  corner      - 'blue' or 'red' (the boxer who received the hit)
  hand        - 'left' or 'right' (the hand that threw the punch)
  screen_x/y  - normalised viewport coordinates [0,1] of the hit location
  world_x/y/z - world-space position of the hit location (F3)
  punch_type  - ExpandedPunchType enum name (e.g. Jab, Cross, LeadHook)
  damage_type - DamageType enum name (e.g. None, Hit, Knockdown)
  weak_point  - DamageWeakPointType enum name; 'None' when no weak point was struck
                (e.g. Nose, Chin, TempleLeft, Liver, SolarPlexus)

punches_thrown.txt  (tab-separated, one row per punch thrown)
  round_time  corner  hand  punch_type
  round_time  - elapsed seconds when the punch was thrown (F2)
  corner      - 'blue' or 'red' (the boxer who threw the punch)
  hand        - 'left' or 'right'
  punch_type  - ExpandedPunchType enum name
  Cleared at the start of each round. Logs every completed punch regardless of whether it landed.

winner.txt  (written when match result is announced; absent if no result yet)
  corner  [TAB]  player_name
  corner      - 'blue', 'red', or 'draw'
  player_name - display name of the winning player (empty for draw)

scores.csv  (updated after each round; absent until first round ends)
  round,blue_score,red_score,blue_total,red_total,blue_damage_taken,red_damage_taken,blue_kds,red_kds
  round              - 1-based round number
  blue_score         - blue corner score for that round (10-point must system)
  red_score          - red corner score for that round
  blue_total         - cumulative blue score through this round
  red_total          - cumulative red score through this round
  blue_damage_taken  - total damage received by blue this round (F1)
  red_damage_taken   - total damage received by red this round (F1)
  blue_kds           - knockdowns suffered by blue this round (all types)
  red_kds            - knockdowns suffered by red this round (all types)

camera.txt  (tab-separated, updated every poll interval)
  pos_x pos_y pos_z  [TAB]  quat_x quat_y quat_z quat_w  [TAB]  fov
  pos     - world-space camera position (F3)
  quat    - camera rotation as unit quaternion (F4)
  fov     - vertical field of view in degrees (F2)

camera_input.txt  (read every frame when file camera is live; toggle with configured ToggleFileCamera key, default F6)
  pos_x pos_y pos_z  [TAB]  quat_x quat_y quat_z quat_w  [TAB]  fov  [TAB]  hide_blue_head  [TAB]  hide_red_head
  Same pos/rot/fov format as camera.txt. Written once at match start with the
  ring centre position and False for both head flags.
  hide_blue_head / hide_red_head - 'True' or 'False'; hides that boxer's head
    model while keeping the body visible. Both fields are optional (default False).

--- blue/ and red/ ---

name.txt
  Player display name (written once at match start).

cosmetics.txt  (one line per gear slot, written once at match start)
  slot_type : option_id : variant_id : primary_colour : secondary_colour : trim_colour
  Slots: Gloves, Shorts, Shoes, Tops

head_position.txt
glove_left_position.txt
glove_right_position.txt  (space-separated, updated every poll interval)
  screen_x screen_y world_x world_y world_z quat_x quat_y quat_z quat_w
  screen_x/y  - normalised viewport coordinates [0,1]
  world_x/y/z - world-space position (F3)
  quat        - transform rotation as unit quaternion (F4)

punishment_mid.txt
  Single value: mid-range punishment weighted percent [0,1] (F3).

punishment_long_raw.txt
  Single value: long-range punishment normalised meter [0,1] (F3).

punishment_long_weighted.txt
  Single value: long-range punishment weighted percent [0,1] (F3).

portrait.png
  2048x2048 PNG of the boxer, captured once at match start.

--- Root ---

lobby.txt  (present while the lobby screen is open; deleted on close)
  slot_N: type=<PlayerType> occupied=<bool> name=<name> ready=<bool>
    (one line per slot; N is 0-based slot index)
  venue_id: <int>
  rounds: <int>
  round_duration: <seconds>
  break_duration: <seconds>
  ready_to_start: <bool>

--- Accessibility notes ---

Corner labels are plain text ('blue' and 'red'), not colour-only, so overlays
can support colour-blind viewers without guessing which boxer is which.

The weak_point column names the specific body region struck (e.g. Nose, Chin,
Liver). Tools can use this to produce descriptive hit annotations such as
'jab to the chin' for screen-reader or audio-commentary output.

World and screen coordinates are provided together so assistive overlays can
place hit markers accurately at any viewport size or aspect ratio.

--- Keyboard shortcuts (defaults; re-bindable via InputActionsSpectator) ---

  In-match
    F6                toggle file camera on/off

  Lobby (host only)
    F5                start the match
    K + 0/1/2/3       kick the player in lobby slot 0, 1, 2, or 3
    M + S             swap red boxer <-> blue boxer
    M + R             swap first spectator <-> red boxer
    M + B             swap first spectator <-> blue boxer
